# ECR Demo — Docker + GitHub Actions + AWS ECR

A Spring Boot application demonstrating containerisation, secure AWS authentication via GitHub OIDC, and automated image publishing to Amazon ECR.

---

## Architecture

See [ARCHITECTURE.md](ARCHITECTURE.md) for full flow diagrams.

```
Developer Push → GitHub Actions → (OIDC) → IAM Role → Amazon ECR
```

No AWS access keys are stored anywhere. GitHub Actions authenticates with AWS using short-lived OIDC tokens.

## AWS Infrastructure Setup

All AWS resources are defined in [infrastructure.yml](infrastructure.yml). Deploy with:

```bash
aws cloudformation deploy \
  --template-file infrastructure.yml \
  --stack-name ecr-demo-stack \
  --parameter-overrides \
      GitHubOrg=YOUR_GITHUB_USERNAME \
      GitHubRepo=ecr-demo \
  --capabilities CAPABILITY_NAMED_IAM \
  --region us-east-1
```

This creates:
- **ECR Repository** (`eugene_ecr-demo`) with scan-on-push and AES256 encryption
- **GitHub OIDC Identity Provider** in IAM
- **IAM Role** (`GitHub-ECR-Push-Role`) with a least-privilege ECR policy

Retrieve the outputs for use as GitHub secrets:

```bash
aws cloudformation describe-stacks \
  --stack-name ecr-demo-stack \
  --query "Stacks[0].Outputs" \
  --output table
```

## GitHub Repository Secrets

Add these two secrets under **Settings → Secrets and variables → Actions**:

| Secret | Value |
|--------|-------|
| `AWS_ROLE_ARN` | ARN from the `GitHubActionsRoleARN` stack output |
| `AWS_REGION` | `us-east-1` |

No `AWS_ACCESS_KEY_ID` or `AWS_SECRET_ACCESS_KEY` — OIDC handles authentication.

## CI/CD Pipeline

The workflow at [.github/workflows/ecr.yml](.github/workflows/ecr.yml) runs on every push to `main`:

| Step | Action |
|------|--------|
| Checkout | `actions/checkout@v4` |
| Configure AWS Credentials | `aws-actions/configure-aws-credentials@v4` (OIDC) |
| Login to ECR | `aws-actions/amazon-ecr-login@v2` |
| Build JAR | `mvn clean package` |
| Build image | `docker build` |
| Tag image | `docker tag … :latest` |
| Push image | `docker push` |

Any step failure stops the pipeline — no partial pushes.

## IAM Trust Policy

The role generated by the CloudFormation template restricts access to this repository only:

```json
{
  "Condition": {
    "StringEquals": {
      "token.actions.githubusercontent.com:aud": "sts.amazonaws.com"
    },
    "StringLike": {
      "token.actions.githubusercontent.com:sub": "repo:YOUR_ORG/ecr-demo:*"
    }
  }
}
```

## IAM ECR Policy (Least Privilege)

```json
{
  "Statement": [
    {
      "Effect": "Allow",
      "Action": ["ecr:GetAuthorizationToken"],
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": [
        "ecr:BatchCheckLayerAvailability",
        "ecr:CompleteLayerUpload",
        "ecr:InitiateLayerUpload",
        "ecr:PutImage",
        "ecr:UploadLayerPart",
        "ecr:BatchGetImage"
      ],
      "Resource": "arn:aws:ecr:us-east-1:ACCOUNT_ID:repository/eugene_ecr-demo"
    }
  ]
}
```
